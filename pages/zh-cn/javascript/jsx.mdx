## 什么是 JSX？

JSX 是 JavaScript 语言的一个扩展，它增加了一个新的 **JSX 表达式**，<Details>用来创建 React 元素</Details>。


JSX表达式是 `React.createElement(type, props, ...children)` API 的简洁语法，我们可以在任何可使用表达式的地方使用 JSX，如 `return` 语句或变量赋值。

---

## 类型

和 XML 一样，JSX 是用标签来表示的。例如 `<View />` 这个标签会被转化成 `React.createElement(View)`， 用来创建一个类型为 `View` 的元素。

<Example panes={['editor', 'transpiler']} code={`const element = <View />`} />

> 点击上述示例的右栏，你可以看到 JSX 表达式是如何被 Babel 转换为 JavaSript 代码。

---

## Props 参数

JSX 里的属性都将成为 React 元素的 **props** 参数。属性的数据类型可以为字符串，如 `foo="hello"，也可以是任何用大括号括起来的 JavaScript 表达式，例如在 `bar = {baz}` 中，会将 `bar` 属性的值设置为 `baz`。

<Example
  panes={['editor', 'transpiler']}
  code={`const element = <View foo="hello" bar={baz} />`}
/>

---

## 子元素
任何子元素都应该位于开始标签 `<View>` 和结束标签 `</View>` 之间。如果没有子元素的话，可以使用自闭标签如 `<View />` 让代码更简练。

<Example panes={['editor', 'transpiler']} code={jsx} />

> 在分行展示 JSX 的使用，一般会加上括号来增加可读性。

---

## 插值

我们也可以用插值的方式（用大括号来包含表达式，如 `{...}`）来加入子元素。

<Example panes={['editor', 'transpiler']} code={jsxChildren} />

---

## 为什么使用 JSX？

类似 XML 的语法让代码更简洁易读，而且其树状结构在外观上和生成的 UI 类似。

React 并不强制要求使用 JSX， 但不用它的弊大于利，因此大多数人还是会选择使用它。
